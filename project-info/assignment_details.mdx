Lila Backend Assignment: Multiplayer Tic-Tac-Toe
Overview
Lila Engineering is driven by ownership and problem-solving. They are currently building technically complex multiplayer shooters and are looking for developers with strong fundamentals capable of building solutions end-to-end.
+1

Problem Statement
Build a production-ready, multiplayer Tic-Tac-Toe game with a server-authoritative architecture using Nakama as the backend infrastructure.

Technical Requirements

1. Frontend

Tech Stack: Choose your preferred stack (React, React Native, Flutter, Unity, etc.).

UI: Implement a responsive UI optimized for mobile devices.

Features:

Display real-time game state updates.

Show player information and match status.

2. Backend (Nakama)

Server-Authoritative Game Logic:

Implement all game state management on the server side.

Validate all player moves server-side before applying them.

Prevent client-side manipulation or cheating.

Broadcast validated game state updates to connected clients.

Matchmaking System:

Enable players to create new game rooms.

Implement automatic matchmaking to pair players.

Support game room discovery and joining.

Handle player connections and disconnections gracefully.

3. Deployment

Server: Deploy Nakama server to a cloud provider (AWS, GCP, Azure, DigitalOcean, etc.).

Client: Deploy the frontend with public accessibility or share the mobile application.

Documentation: Provide clear deployment documentation.

Optional Bonus Features
[ ] Concurrent Game Support: Handle multiple simultaneous game sessions with proper room isolation and scalable architecture.

[ ] Leaderboard System: Track wins, losses, and streaks; implement a global ranking system with persisted data.
+1

[ ] Timer-Based Game Mode: Add turn time limits (e.g., 30 seconds), automatic forfeit on timeout, and mode selection (classic vs. timed).
